# MediFootcare.web — UI/UX・ユーザージャーニーレビュー
作成日：2026-03-27

---

## プロジェクト概要

- **フレームワーク**: Next.js 15.2.4 / React 19 / TypeScript
- **スタイル**: Tailwind CSS + Material Symbols
- **デプロイ**: GitHub Pages（静的エクスポート）
- **ターゲット**: 個人（高齢者・家族）と施設担当者の2セグメント

---

## ユーザーの主要ゴール定義

| セグメント | 主要ゴール | 成功指標 |
|---|---|---|
| 個人（家族含む） | 親・本人の足の悩みを解決するサービスを信頼して予約する | フォーム送信 or LINE接触 |
| 施設担当者 | 施設への定期導入可否を判断し、試験導入に進む | 問い合わせ → 体験会申し込み |

---

## 理想的なユーザーフロー（ステップ単位）

### 個人ユーザー

```
1. 流入（検索・SNS）
2. Hero で「自分向けか」を 0〜3秒で判断
3. 「こんな悩みがある」と共感（Problems セクション）
4. 「信頼できるか」を確認（資格・感染対策）
5. 「料金・流れ」を把握（個人ページへ）
6. 「対応エリアか」確認
7. 問い合わせ（フォーム or LINE）
```

### 施設担当者

```
1. 流入（営業資料・口コミ）
2. Hero の「施設向けご案内」をクリック
3. 施設ページで「導入メリット」を確認
4. 「最小何名から？」「書類は？」を FAQ で確認
5. 体験会ボタンを押す
6. 問い合わせ
```

## 現状とのギャップ分析

### P0｜最大のギャップ：コンバージョンポイントが機能していない

**問題**：フォームURLとLINEリンクが未実装のまま本番コードに残っている。

```
// 全ページのCTAボタン
href="https://docs.google.com/forms/d/e/DUMMY_FORM_ID/viewform"  ← 壊れたURL
href="#"  ← LINE ボタン（全箇所）
```

**影響箇所**：
- Header の「お問い合わせ」ボタン
- Homepage のお問い合わせセクション（フォーム・LINE 各ボタン）
- personal/page.tsx の PersonalCTA コンポーネント（2箇所）
- facility/page.tsx の FacilityCTA コンポーネント（2箇所）
- Footer の「お問い合わせフォーム」リンク

**影響**：問い合わせに到達したユーザーが全員離脱する。他の改善はすべて無意味になる致命的な欠陥。

---

### P0｜モバイルナビゲーションが存在しない

**問題**：Header.tsx でモバイル時にナビゲーションを `hidden md:flex` で非表示にしているが、ハンバーガーメニュー等の代替手段が実装されていない。

```tsx
// Header.tsx:21
<nav className="hidden md:flex ...">  ← モバイルでは完全に消える
```

**影響**：スマートフォンユーザーが個人ページ・施設ページ・FAQ へ回遊できない。Hero の CTA ボタン以外の導線がない。医療系サービスは高齢者の家族（スマホユーザー）が主な流入元であり、影響が大きい。

---

### P0｜Hero のボタンリンクが localhost を参照している

**問題**：ホームページの Hero セクションのボタンが開発環境の URL をハードコードしている。

```tsx
// app/page.tsx:52-56
href="http://localhost:3000/medifootcare.web/facility"
href="http://localhost:3000/medifootcare.web/personal"
```

**影響**：本番環境でボタンを押してもページ遷移できない（localhost に飛ぼうとする）。

---

### P1｜Hero のデュアル CTA が認知コストを生む

**問題**：ページ到達直後に「施設向け」「個人向け」を並列で提示している。ユーザーは「足のケアをしてほしい」という状態で来訪しており、いきなりセグメント選択を求められると戸惑う。また施設と個人が同じスタイルのボタンで表示されており、優先度の差がない。

**現状のボタン順**：施設向け（左）→ 個人向け（右）
※個人ユーザーの方が件数的に多いと思われるが、左（先）に施設向けが来ている。

---

### P1｜料金の発見タイミングが遅い

**問題**：ホームページのセクション順序が以下のようになっており、料金は6番目。

```
1. Hero
2. お悩み（Problems）
3. 5つの特徴
4. ケース紹介
5. 安心ポイント
6. 料金案内  ← ここまでスクロールが必要
7. よくある質問
8. 対応エリア
9. お問い合わせ
```

**影響**：料金は意思決定の最重要情報であるにもかかわらず、スクロール量が多い。特に高齢の家族向けユーザーは途中で離脱しやすい。

---

### P1｜施設ページの「最低5名」情報が FAQ に隠れている

**問題**：施設担当者が最初に確認したい「うちの施設は対象か？」という条件が、FAQ（継続訪問についてセクション）を開かないと分からない。

```tsx
// facility/page.tsx:169
{ q: "何名から依頼できますか？", a: "5名以上から対応可能です。" }
```

**影響**：小規模施設の担当者が問い合わせてから「対象外」と判明し、双方の工数が無駄になる。または最初から諦めて離脱する。

---

### P2｜対応エリアの表現（対応済みに近い）

**現状**：エリアセクションに「上記以外のエリア（近隣市区町村）もお気軽にご相談ください。」という記載がある。ただしトップのよくある質問の回答でも「佐倉市・八千代市・四街道市・印西市ほか近隣」と書かれており、一貫性はある。
→ 大きな問題ではないが、ページ最上部（Hero 直下など）にも一言添えるとよい。

---

### P3｜ケースページに画像欠損がある

**問題**：「角質・かかとのひび割れ」のみ Before/After 画像がなく、他の5ケースと視覚的一貫性が崩れる。

---

### P3｜信頼シグナルがテキストのみ

**問題**：「看護師資格保有」と書いてあるが、資格証・バッジの画像・登録番号などの視覚的証拠がない。医療系サービスとして信頼感を高めるには根拠の可視化が重要。

---

### P4｜口コミ・実績数がない

**問題**：Before/After 画像はあるが、施術件数・口コミ・評価のソーシャルプルーフがない。「何件施術してきたのか」が不明。

---

## 離脱・迷い・不信感が発生するポイント

| 深刻度 | 発生箇所 | 種別 | 理由 |
|---|---|---|---|
| 🔴 致命的 | フォームCTAボタン押下時 | 離脱 | URLが壊れており404またはエラー |
| 🔴 致命的 | LINE ボタン押下時 | 離脱 | `href="#"` で反応なし |
| 🔴 致命的 | Hero の個人・施設ボタン押下時 | 離脱 | localhost URL で本番では遷移失敗 |
| 🔴 致命的 | モバイルでの回遊 | 迷い | ナビゲーション消滅、行き先不明 |
| 🟠 高 | Hero のデュアル CTA | 迷い | セグメント選択を即求められる |
| 🟠 高 | 施設ページの最低人数 | 不信感 | 主要条件が FAQ に隠れている |
| 🟠 高 | 料金の発見タイミング | 離脱 | スクロールが長く離脱しやすい |
| 🟡 中 | ケースページの画像欠損 | 不信感 | コンテンツの一貫性が崩れる |
| 🟡 中 | 資格・クレデンシャルの詳細なし | 不信感 | テキストのみで証明がない |
| 🟡 中 | 口コミ・実績数の不在 | 不信感 | ソーシャルプルーフがない |

## URL 一元管理の推奨構成

現在フォーム URL・LINE URL が各ファイルに散在しているため、定数ファイルを作成して一括管理することを推奨。

```
// constants/links.ts
export const FORM_URL = "https://docs.google.com/forms/d/e/YOUR_FORM_ID/viewform";
export const LINE_URL = "https://lin.ee/XXXXXXX";
```

差し替えが必要な箇所：
- components/Header.tsx（1箇所）
- components/Footer.tsx（1箇所）
- app/page.tsx（2箇所）
- app/personal/page.tsx - PersonalCTA コンポーネント（2箇所）
- app/facility/page.tsx - FacilityCTA コンポーネント（2箇所）

## 総評

デザイン品質・コンテンツ設計・情報構造は高水準。医療系らしい信頼感、個人/施設の明確な分離、アコーディオン FAQ など、プロダクトの骨格は完成している。

**ただし現時点では「動かないサイト」である。**

P0 の3点（フォーム URL・localhost URL・モバイルナビ）を修正するだけで、ユーザーの到達率と CV 率が大幅に改善する。まずここを最優先に対応すること。
